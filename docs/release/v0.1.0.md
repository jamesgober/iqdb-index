# iqdb-index v0.1.0 — Scaffold

**The foundation is laid.** v0.1.0 is the repository bootstrap for `iqdb-index`: the manifest, the dual license, the documentation skeleton, the lint configuration, and the CI matrix that every later release is built and verified against. There is no domain logic yet — this release establishes the structure and the quality gates, nothing more. The `Index` trait, `IndexStats`, and the batch defaults land in v0.2.0.

## What is iqdb-index?

The polymorphism layer of the iQDB vector database. The engine calls `index.search(query, params)` and the right implementation runs, so iQDB supports plug-and-play index strategies instead of being hardcoded to one. It is almost entirely trait definitions plus a small amount of shared utility, and it depends only on `iqdb-types`.

## What's in 0.1.0

- **Manifest** — `Cargo.toml` with crate metadata, Rust 2024 edition, MSRV 1.87.
- **License** — dual `Apache-2.0 OR MIT` (`LICENSE-APACHE`, `LICENSE-MIT`).
- **Docs skeleton** — `README.md`, `CHANGELOG.md` (Keep a Changelog 1.1.0), and `docs/API.md` sketching the tiered surface to come.
- **Standards** — `REPS.md` compliance baseline, `dev/DIRECTIVES.md` (engineering standards + definition of done), and `dev/ROADMAP.md` (the path to 1.0).
- **Tooling** — `.github/workflows/ci.yml` (Linux/macOS/Windows × stable/1.87, plus loom and security jobs), `deny.toml`, `clippy.toml`, `rustfmt.toml`.
- **Lints** — the crate-level lint header and `#![forbid(unsafe_code)]` in `src/lib.rs`.

## Breaking changes

**None.** This is the first release.

## Verification

Built and tested on Windows x86_64 (Rust stable 1.95.0). The CI matrix runs the same across Linux, macOS, and Windows at both `stable` and the `1.87` MSRV floor:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
```

All green. The scaffold compiles clean and the single smoke test passes; there is no domain surface to exercise yet.
