# iqdb-index v0.5.0 — API Frozen

**The surface is locked.** v0.5.0 freezes the public API of `iqdb-index` for the 1.x series and records each implementation's deletion semantics against the trait's deletion contract. From here, only additive, non-breaking changes ship until 2.0. This is the last design milestone before the RC track.

## What is iqdb-index?

The polymorphism layer of the iQDB vector database: the `IndexCore` operational trait, the `Index` typed-construction trait, and `IndexStats`. The engine holds indexes as `Box<dyn IndexCore>` and runs the right strategy without being hardcoded to one. Its only dependency is `iqdb-types`.

## What's new in 0.5.0

### Per-implementation deletion semantics

The trait's deletion contract is specified by *observable behaviour* — "a deleted id never reappears in `search` until re-inserted" — not by a storage mechanism. v0.5.0 records how each real consumer reaches that result:

| Crate | Mechanism | Storage | `len()` |
|---|---|---|---|
| `iqdb-flat` | **true removal** | reclaimed immediately (`swap_remove`) | drops by 1 |
| `iqdb-hnsw` | **tombstone** | retained (node kept for graph connectivity) | drops by 1 (live count) |
| `iqdb-ivf` | **true removal** | reclaimed (`swap_remove` from the cluster's posting list) | drops by 1 |

That flat and ivf reclaim while hnsw tombstones — all behind one unchanged contract — is exactly why the trait specifies behaviour, not mechanism. A graph index cannot excise a node without breaking connectivity, so it tombstones; a flat or clustered index drops a row in O(1). The table now lives in `docs/API.md`.

### API freeze

The public surface is **frozen for 1.x** and recorded in `dev/ROADMAP.md`:

- `IndexCore` — the eight required methods plus the three provided shims (`insert_batch`, `search_batch`, `is_empty`), `Send + Sync`.
- `Index` — associated `Config: Default + Clone` and `new(dim, metric, config)`.
- `IndexStats` — the five public fields.
- Empty feature set.

The deliberate freeze decisions are recorded too: the `IndexCore` / `Index` split (object safety), the synchronous design (v0.4.0), the `Arc<[f32]>` shared payload, and that `IndexStats` is **not** `#[non_exhaustive]` (new per-kind detail goes in `extra`, never in a new field). `cargo audit` and `cargo deny check` are clean.

### More examples

The example set now covers four distinct facets of the surface:

- [`custom_index`](../../examples/custom_index.rs) — implement `IndexCore` + `Index` by hand (Tier 3).
- [`polymorphic_engine`](../../examples/polymorphic_engine.rs) — hold several index kinds as `Box<dyn IndexCore>` and dispatch.
- [`dot_product_ordering`](../../examples/dot_product_ordering.rs) — the `DotProduct` negation contract (store `-dot` so "most similar" sorts first).
- [`batch_and_stats`](../../examples/batch_and_stats.rs) — the default `insert_batch` / `search_batch` shims and `IndexStats::extra` (a tombstone counter).

## Breaking changes

**None.** No code or public-surface change from v0.4.0; this release is the freeze commitment plus documentation and examples.

## Verification

Run on Windows x86_64 (Rust stable 1.95.0). The same commands run in the CI matrix across Linux, macOS, and Windows at both `stable` and the `1.87` MSRV floor:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps --all-features
cargo deny check
cargo audit
```

All green. **30 tests** (16 trait-shape + 5 property + 3 `IndexStats` + 3 consumer-simulation) + 3 doctests, plus 4 runnable examples. `cargo deny check` reports `advisories ok, bans ok, licenses ok, sources ok`; `cargo audit` finds zero advisories. Zero `unsafe` (`#![forbid(unsafe_code)]`).

## What's next

- **v0.6.0 → v0.9.x — Alpha / Beta → RC.** Integrate against the live consumers (`iqdb-flat`, `iqdb-hnsw`, `iqdb-ivf`, then `iqdb-build` / `iqdb-eval` / `iqdb`), MINOR-compatible additions only, broader testing, final review. This phase needs those crates published and compiling against the frozen `0.5.x` surface.
- **v1.0.0 — Stable**, once the consumers have soaked against the frozen API.
